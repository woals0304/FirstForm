# 강호 수련록

Unity 기반 2D 세로형 무협 로그라이트 MVP 프로토타입입니다.

완성 게임이 아니라 핵심 루프와 데이터 흐름을 확인하기 위한 개발용 프로젝트입니다.

## 핵심 루프

1. 입문 무공 선택
2. 수련
3. 강호 출행
4. 탐험
5. 자동 전투
6. 전투 승리 또는 사망
7. 전투 승리 시 보상 확인 및 다음 행동 선택
8. 사망 시 혼백 상태와 새 육신 선택
9. 새 회차 시작

## 현재 구현된 기능

- 런타임 자동 생성 세로형 UI
- 입문 무공 3종 선택
  - 청풍검식
  - 파문검식
  - 회류보
- 자동 수련
  - 검법 숙련도, 내력, 근력 증가
  - 일정 시간 후 자동 출행
- 간단한 탐험 단계
- 자동 전투
  - 플레이어/적 자동 공격
  - 적 강공 예고
  - 회피, 막기, 집중, 강행돌파 대응
- 전투 승리 상태
  - 영혼 성장 포인트 획득
  - 전투 승리 횟수 증가
  - 임시 전리품 표시
  - 계속 출행 / 수련지 복귀 선택
- 계속 출행 난이도 증가
  - 출행 단계가 오를수록 적 체력과 공격력이 조금 증가
- 사망 처리
  - 사망 후 육신 후보 3개 생성
  - 선택한 육신 보너스로 새 회차 시작
- 혼백 성장
  - 혼의 맷집
  - 잔류 검의
  - 맑은 내력
- 최소 저장/불러오기
- 개발용 Debug Control

## 실행 방법

1. Unity에서 이 프로젝트 폴더를 엽니다.
2. `Assets/Scenes/SampleScene`을 엽니다.
3. Play 버튼을 누릅니다.
4. 씬에 UI가 없으면 런타임에 기본 세로형 UI가 자동 생성됩니다.

씬에 `GameManager`가 없어도 런타임에서 기본 게임 루트가 자동 생성되도록 되어 있습니다.

## Debug Control

개발 테스트용 Debug Control은 화면의 `DEBUG` 버튼으로 접고 펼칠 수 있습니다.

현재 버튼:

- 전투: 즉시 전투 시작
- 사망: 플레이어 즉시 사망
- 육신: 육신 선택 상태로 이동
- 회복: 플레이어 체력/내력 회복
- 적HP1: 현재 적 체력을 1로 변경
- 무공초기화: 입문 무공 선택 상태로 되돌림
- 저장: 현재 진행 저장
- 불러오기: 저장 데이터 불러오기
- 초기화: 저장 데이터 초기화

혼백 성장 버튼은 Debug Control과 분리되어 있습니다.

- 맷집+
- 검의+
- 내력+

`UIManager`의 `showDebugControls` 값을 끄면 Debug Control UI는 숨겨집니다.

## 폰트 설정

자동 생성 UI는 TextMeshProUGUI를 사용합니다.

기본 TMP 폰트에는 한글 글리프가 없을 수 있으므로 한글 TMP Font Asset 연결을 권장합니다.

설정 방법:

1. 한글을 지원하는 `.ttf` 또는 `.otf` 폰트를 `Assets/Art/Fonts/` 같은 폴더에 넣습니다.
2. Unity에서 TMP Font Asset을 생성합니다.
3. `UIManager`의 `Runtime UI Font > Korean Tmp Font` 슬롯에 연결합니다.
4. Play 시 자동 생성되는 TMP 텍스트에 해당 폰트가 적용됩니다.

폰트를 연결하지 않아도 실행은 계속되지만, 한글이 네모로 보일 수 있습니다.

## 저장 시스템

저장은 `PlayerPrefs`에 `SaveData` JSON 문자열로 저장합니다.

현재 저장되는 주요 정보:

- 선택한 입문 무공
- 현재 회차
- 현재 육신 이름
- 영혼 성장 포인트
- 혼백 성장 레벨
- 총 사망 횟수
- 총 전투 승리 횟수

자동 저장 시점:

- 입문 무공 선택 후
- 전투 승리 보상 획득 후
- 사망 보상 획득 후
- 새 육신 선택 후 새 회차 시작 시
- 혼백 성장 강화 후
